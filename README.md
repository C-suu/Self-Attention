# Self-Attention 手撕 基于`torch`
### （1）题目含义与生僻概念白话文解释

**题目含义**
这段代码利用PyTorch深度学习框架，实现了一个标准的“自注意力机制（Self-Attention）”模块。自注意力机制是Transformer模型（大语言模型的核心基础）的关键组件。其主要作用是处理序列数据时，让序列里的每一个元素都能“看到”并“关注”到序列中的其他所有元素，从而计算出每个元素在当前上下文中的最优表示。

**生僻概念解释**
*   **Query (Q/查询)**：可以理解为“当前正在寻找匹配项的特征”。就像在图书馆的检索系统里输入的检索词。
*   **Key (K/键)**：可以理解为“其他元素身上自带的标签”。就像图书馆里每一本书的关键词。
*   **Value (V/值)**：可以理解为“其他元素本身的实质内容”。就像图书馆里那本书的具体内容。
*   **Self-Attention 匹配过程**：系统会拿当前元素的 `Query`，去和所有元素的 `Key` 计算相似度（点积运算）。如果两者的相似度极高，说明当前元素需要高度关注那个元素。系统会根据这个相似度算出一个“关注度比例”（即注意力权重），然后按这个比例把所有元素的 `Value` 混合起来，作为当前元素最终的特征表达。

### （2）代码视角的解题思路

从代码的执行流转来看，解题思路可以拆解为以下几个严密的数学与张量操作步骤：

1.  **初始化权重层**：在类初始化阶段，提前定义好三个线性变换矩阵（`linear_q`, `linear_k`, `linear_v`）。这三个矩阵的作用是将输入特征映射到对应的Q、K、V三个不同特征空间。同时预先计算好一个缩放因子（`_norm_fact`），用以稳定后续的梯度计算。
2.  **维度校验**：在数据向前传播（`forward`）的开始，提取输入张量 `x` 的维度大小，并断言其特征维度与初始化的预期维度完全一致，确保矩阵乘法能够顺利进行。
3.  **空间映射**：利用第一步构建的三个线性变换层，分别对输入张量 `x` 进行处理，得到三组全新的张量：查询张量 `q`、键张量 `k` 和值张量 `v`。
4.  **计算原始相关性得分**：利用批量矩阵乘法（`torch.bmm`），将 `q` 张量与转置后的 `k` 张量相乘。其本质就是让所有的查询特征与所有的键特征进行点积，得出的结果矩阵代表了元素两两之间的原始关联强度。
5.  **数值缩放与概率化**：将关联强度矩阵乘以事先算好的缩放因子，防止内积数值过大导致后续计算溢出或梯度消失。随后在最后一个维度上施加 `softmax` 操作，将无界的原始得分转化为总和为1的概率分布（注意力权重矩阵）。
6.  **特征融合（加权求和）**：最后，再次利用批量矩阵乘法，将得到的概率分布矩阵（注意力权重）与值张量 `v` 相乘。这就完成了“按照权重提取相关信息”的核心逻辑，得到的结果直接输出。

### （3）带有详细注释的代码框

```python
from math import sqrt                 # 01. 从数学库导入平方根计算函数
import torch                          # 02. 导入PyTorch张量运算基础库
import torch.nn as nn                 # 03. 导入PyTorch神经网络核心模块

class SelfAttention(nn.Module):       # 04. 定义自注意力类，继承自神经网络基础类
    def __init__(self, dim_q, dim_k, dim_v): # 05. 初始化方法，接收输入维度、键维度、值维度
        super(SelfAttention, self).__init__()# 06. 调用父类的初始化方法
        self.dim_q = dim_q            # 07. 将输入特征的维度保存为类的属性
        self.dim_k = dim_k            # 08. 将键(Key)和查询(Query)的目标映射维度保存
        self.dim_v = dim_v            # 09. 将值(Value)的目标映射维度保存
        
        # 10. 定义Query的全连接映射层，不使用偏置项
        self.linear_q = nn.Linear(dim_q, dim_k, bias=False) 
        # 11. 定义Key的全连接映射层，不使用偏置项
        self.linear_k = nn.Linear(dim_q, dim_k, bias=False) 
        # 12. 定义Value的全连接映射层，不使用偏置项
        self.linear_v = nn.Linear(dim_q, dim_v, bias=False) 
        # 13. 计算并保存缩放因子：Key维度的平方根的倒数
        self._norm_fact = 1 / sqrt(dim_k)                   

    def forward(self, x):             # 14. 定义前向传播函数，x为输入张量
        batch, n, dim_q = x.shape     # 15. 获取输入张量的批次大小、序列长度及特征维度
        assert dim_q == self.dim_q    # 16. 断言检查：确保输入的特征维度与初始化设定一致
        
        q = self.linear_q(x)          # 17. 将输入映射为Query矩阵，形状: (batch, n, dim_k)
        k = self.linear_k(x)          # 18. 将输入映射为Key矩阵，形状: (batch, n, dim_k)
        v = self.linear_v(x)          # 19. 将输入映射为Value矩阵，形状: (batch, n, dim_v)
        
        # 20. 计算注意力得分：q乘上转置后的k，并进行缩放。结果形状: (batch, n, n)
        dist = torch.bmm(q, k.transpose(1, 2)) * self._norm_fact  
        # 21. 对最后一个维度(n)执行Softmax归一化，生成注意力权重分布
        dist = torch.softmax(dist, dim=-1)                        
        
        att = torch.bmm(dist, v)      # 22. 权重矩阵与Value矩阵相乘进行特征聚合
        return att                    # 23. 返回最终自注意力处理完毕的张量
```

### （4）每一行代码详细的中文解释

*   **第01行**：从内置的 `math` 模块中引入计算平方根的 `sqrt` 函数，后续将用于计算缩放常数。
*   **第02行**：导入 `torch` 基础包，提供了多维张量(Tensor)数据结构和相应的数学运算支持。
*   **第03行**：导入 `torch.nn` 模块并重命名为 `nn`，这里面包含了构建神经网络所需的各类标准层（如全连接层等）。
*   **第04行**：声明一个名为 `SelfAttention` 的类，它继承了 `nn.Module`。这是在PyTorch中自定义网络模型的标准范式。
*   **第05行**：构建初始化构造函数，该函数需要外界在实例化时提供三个参数：查询特征输入维度 `dim_q`、内部键维度 `dim_k` 以及内部值维度 `dim_v`。
*   **第06行**：执行继承自 `nn.Module` 父类的构造逻辑，以使得当前类获得网络层级管理、参数注册等基本能力。
*   **第07行**：把传入的 `dim_q` 参数保存到实例自身的变量 `self.dim_q` 中，以便在后续步骤中随时提取比对。
*   **第08行**：把传入的 `dim_k` 参数保存到实例属性中。通常 Query 和 Key 需要进行点积操作，因此映射后的维度必须保证完全一致。
*   **第09行**：把传入的 `dim_v` 参数保存到实例属性中，该参数决定了输出特征的向量宽度。
*   **第10行**：实例化一个线性映射层（无偏置），负责输入特征经过矩阵相乘后，转化为 Query (Q) 张量。
*   **第11行**：实例化第二个线性映射层（无偏置），其结构与Q层的输入输出规格一致，专门负责生成 Key (K) 张量。
*   **第12行**：实例化第三个线性映射层（无偏置），负责将输入特征转换为 Value (V) 张量，输出维度变为 `dim_v`。
*   **第13行**：计算注意力机制的标准化缩放系数。提取Key特征维度的数值进行开方，随后求其倒数。
*   **第14行**：定义前向计算逻辑，该方法会在传入数据流经过本网络模块时自动触发运行，`x` 为外界输入的三维张量矩阵。
*   **第15行**：利用拆包语法，将输入张量 `x` 的形状（Shape）分别赋值给 `batch`（样本批次数）、`n`（时间步或序列元素数量）、`dim_q`（单个元素的特征宽度）。
*   **第16行**：实施安全性校验。利用 `assert` 强制检验此时流入的数据宽度 `dim_q` 是否等于类被实例化时登记的期待宽度 `self.dim_q`。如果不等，程序会直接在此处报错终止。
*   **第17行**：把输入 `x` 灌入 Query 映射层。此操作会对序列里的每一个向量做线性变换，产出的张量命名为 `q`。
*   **第18行**：把输入 `x` 灌入 Key 映射层，生成供匹配查验使用的目标键张量 `k`。
*   **第19行**：把输入 `x` 灌入 Value 映射层，生成承载实际内容值域特征的张量 `v`。
*   **第20行**：这是核心数学操作。先调用 `transpose(1, 2)` 将 `k` 的序列维度与特征维度互换（转置）。再使用 `torch.bmm` 执行批量的矩阵乘法，得出原始的序列互相关联矩阵。最终在此矩阵上逐元素乘以之前备好的常量因子 `_norm_fact`，防止后续计算中 Softmax 的偏导数趋近于零。
*   **第21行**：针对刚算出的缩放得分矩阵，调用 `torch.softmax`。参数 `dim=-1` 指示在最后一个维度（即每一个Query针对所有Key的相似度切片上）计算指数归一化操作，使得权重皆处于0到1之间，且合计等于1。
*   **第22行**：执行加权聚合操作。使用批量矩阵乘法把注意力权重矩阵 `dist` 和包含了具体信息的特征矩阵 `v` 进行相乘。该乘法本质是对所有的特征行执行按照概率权重的加权求和。
*   **第23行**：将经过注意力加权提取并融合完成后的新特征张量返回给调用方，完成整个模块的生命周期。
